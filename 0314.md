注意新训练和评估脚本不依赖.ratio的cache！！！


Step 1: 生成城市对特征 cache（~7秒/21年）

cd /data1/wxj/Recall_city_project/ && uv run generate_city_pair_cache.py
依赖: data/cities_2000-2020/cities_YYYY.jsonl + data/city_edges.jsonl
输出: data/city_pair_cache/city_pairs_YYYY.parquet（每个 ~4MB，共 21 个）
每年 113,232 对 × 51 个特征（30 比率 + 11 差值 + 6 绝对 + 4 网络距离）
Step 2: 重建精简 feather（~80秒/21年）

uv run rebuild_feather.py
依赖: output/processed_ready/processed_YYYY.feather
输出: output/base_ready/base_YYYY.feather（每个 ~38MB vs 原 583MB，缩小 15 倍）
只保留: qid, From_City, To_City, 6 人口统计, Rank, Label, Year
Step 3: 训练

uv run simple_train.py
依赖: output/base_ready/ + data/city_pair_cache/
输出: output/models/model_v2.txt + checkpoints
动态 join cache + 构建 6 个交叉特征，总计 63 个特征（原 18 个）
Step 4: 评估

uv run evaluate.py --model output/models/model_v2.txt --years 2019 2020
从 simple_train.py import 特征定义，保证对齐
同样的 base + cache join + 交叉特征流程
